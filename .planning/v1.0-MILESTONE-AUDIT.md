### GAP-01: CLI entry point not installed in Docker

- **From:** Phase 3 (`pyproject.toml` entry point `imgproc = "app.cli:app"`)
- **To:** Docker container runtime
- **Issue:** Dockerfile does not copy `pyproject.toml` nor run `pip install -e .`, so `imgproc` command is not in PATH inside the container
- **Impact:** LOW — the container's CMD uses `uvicorn` directly for the HTTP service. CLI is primarily for local use. Workaround: `python -m app.cli <command>` inside container
- **Affected requirements:** CLI-01 through CLI-04 (inside Docker only)
- **Fix:** Add `COPY pyproject.toml .` and `RUN pip install -e . --no-deps` to Dockerfile
